# План: Заполнить все компании по RESEARCH_GUIDE.md

Заполнить 16 карточек компаний-заглушек полным фундаментальным анализом по методологии RESEARCH_GUIDE.md. Для каждой компании: собрать финансовые данные (smart-lab), оценить бизнес, рассчитать справедливую цену, заполнить карточку, обновить индексные файлы.

- **Files involved:**
  - 16 stub-файлов `companies/{TICKER}/_index.md`
  - `companies/_index.md` — обновление sentiment/position в секторных таблицах
  - `_index.md` — добавление строк в таблицу «Статус обновлений»
  - `sectors/{sector}/_index.md` — обновление данных по компаниям в секторах
- **Related patterns:** Заполненные карточки (SBER, GAZP, LKOH и др.) как эталон формата
- **Dependencies:** WebFetch для smart-lab.ru, макро-данные из `russia/macro.md`

## Подход

- Компании сгруппированы по секторам — заполняем сектор за сектором
- Для каждой компании строго следуем шагам 1-9 из RESEARCH_GUIDE.md
- Основной источник данных: smart-lab.ru (годовые + квартальные МСФО)
- После заполнения каждой компании: обновить `companies/_index.md`, `_index.md`, секторальную таблицу
- После заполнения всех компаний: запустить `python3 scripts/generate_trend_json.py`
- Коммит + push после каждого сектора

## Алгоритм для каждой компании

Строго по RESEARCH_GUIDE.md, шаги 1-9:

1. **Прочитать контекст**: секторальный анализ, макро, геополитика, валюта, регулирование
2. **Собрать данные**: WebFetch smart-lab (годовые МСФО + квартальные МСФО), проверить ликвидность
3. **Понять бизнес**: что продаёт, кому, драйверы роста, конкурентные преимущества
4. **Оценить финансовое здоровье**: красные/зелёные флаги, долг, маржа, FCF
5. **Определить справедливую цену**:
   - Шаг 1: EPS, EBITDA per share, Net Debt per share
   - Шаг 2: Целевой P/E = 1/(r-g) или EV/EBITDA для закредитованных
   - Шаг 3: Поправка за качество + GOD-дисконт (если госкомпания)
   - Шаг 4: Три сценария (пессимистичный / базовый / оптимистичный)
   - Шаг 5: Чувствительность к g
   - Шаг 6: Перекрёстная проверка (DCF + дивидендный)
6. **Рассчитать upside, определить position** (buy/watch/hold/sell; avoid — если сработали stop-условия)
7. **Заполнить карточку**: YAML-метаданные + все секции (бизнес-модель, конкурентные преимущества, риски, финпоказатели, дивиденды, оценка, тезис, целевая цена, чеклист, история решений, источники)
8. **Обновить связанные документы**: companies/_index.md, _index.md, sectors/{sector}/_index.md
9. **Обновить trend.json**: запустить скрипт после всех компаний
