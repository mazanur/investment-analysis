# Инструкция по исследованию компании

Цель — определить, недооценена ли компания рынком, и стоит ли в неё инвестировать.

## Порядок работы

### 1. Прочитай контекст

Перед исследованием конкретной компании прочитай **снизу вверх**:

1. `sectors/{sector}/_index.md` — стадия цикла, драйверы, риски, критерии отбора из «Инвестиционного вывода»
2. `russia/macro.md` — ставка ЦБ, инфляция, прогноз (влияет на ставку дисконтирования и переоценку рынка)
3. `russia/geopolitics.md` — военный конфликт, санкции, сценарии развития (влияет на дисконт к оценке)
4. `russia/currency.md` — если компания экспортёр/импортёр или имеет валютный долг
5. `russia/regulations.md` — санкционный статус, налоговые изменения
6. Существующую карточку `companies/{TICKER}/_index.md` — чтобы не потерять ранее собранные данные

### 2. Собери финансовые данные

**Основной источник — smart-lab.ru:**

- Годовые МСФО: `https://smart-lab.ru/q/{TICKER}/f/y/MSFO/`
- Квартальные МСФО: `https://smart-lab.ru/q/{TICKER}/f/q/MSFO/`

Загружай обе страницы через WebFetch. **Проверяй актуальность:** в таблице смотри дату последнего периода. Если данные старше квартала — сверяй с официальным отчётом на e-disclosure.ru.

Из них извлекай:

| Что брать | Зачем |
|-----------|-------|
| Выручка (4+ лет) | Темп роста, стабильность |
| EBITDA, маржа EBITDA | Операционная эффективность, тренд маржи |
| Чистая прибыль | Основа для P/E |
| FCF | Основа для дивидендов и оценки по FCF yield |
| Долг, Net Debt/EBITDA | Финансовая устойчивость |
| P/E, EV/EBITDA, P/BV | Текущая оценка рынком |
| ROE | Качество бизнеса |
| Дивиденды (DPS) | Доходность, стабильность выплат |
| Buyback | Дополнительный возврат акционерам, влияет на EPS |
| Среднедневной объём торгов | Ликвидность (см. ниже) |

**Определение Net Debt:**

```
Net Debt = Долгосрочные кредиты и займы
         + Краткосрочные кредиты и займы
         + Обязательства по аренде (IFRS 16)
         − Денежные средства и эквиваленты
         − Краткосрочные финансовые вложения (депозиты < 3 мес.)
```

smart-lab считает Net Debt автоматически. При ручном расчёте с e-disclosure — используй эту формулу.

**Проверь ликвидность:**

Неликвидная бумага — это риск: сложно войти/выйти без движения цены. Минимальные требования:

| Параметр | Минимум | Где смотреть |
|----------|---------|--------------|
| Среднедневной объём (ADV) | > 10 млн ₽ | moex.com → карточка инструмента |
| Free-float | > 10% | moex.com, smart-lab |
| Спред bid-ask | < 1% | торговый терминал |

Если ADV < 10 млн ₽ — учитывай discount за неликвидность (-10-20% к целевой цене) или откажись от позиции.

**Альтернативные источники (если smart-lab неполон или устарел):**

| Источник | Что брать | URL |
|----------|-----------|-----|

**Дополнительные источники:**

- IR-страница компании — пресс-релизы, презентации, дивидендная политика
- Аналитика брокеров (BCS, SberCIB, T-Investments) — консенсус-прогнозы, таргеты
- Новостной фон — `"{тикер} результаты {год}"`, `"{тикер} дивиденды {год}"`

### 3. Пойми бизнес

Перед тем как считать мультипликаторы — разберись, на чём компания зарабатывает:

- **Что продаёт** — товары, услуги, подписки, лицензии
- **Кому продаёт** — B2C / B2B / B2G, география (внутренний рынок / экспорт)
- **Откуда рост** — органический (новые клиенты, рост ARPU) или M&A
- **Конкурентные преимущества** — масштаб, бренд, технология, лицензии, регуляторный барьер
- **Зависимости** — от сырья, курса валюты, госзаказа, одного клиента

Источники: IR-презентации, годовой отчёт, секторальный анализ.

### 4. Оцени финансовое здоровье

Красные флаги:

| Сигнал | Что значит |
|--------|-----------|
| Net Debt/EBITDA > 3,5x | Высокий долг, риск при высокой ставке |
| Маржа EBITDA падает 3+ кварталов подряд | Теряет эффективность |
| FCF отрицательный 2+ года | Бизнес не генерирует кэш |
| Прибыль растёт, а FCF падает | Прибыль «бумажная» — проверь капекс и оборотный капитал |
| ROE < стоимости капитала (~16% при текущей ставке) | Бизнес разрушает стоимость |
| Выручка стагнирует, а прибыль растёт | Рост за счёт оптимизации, не масштабируется |
| OCF / Net Income < 0.5 | Прибыль не конвертируется в кэш — проверь качество |
| Большие курсовые разницы в P&L | Волатильная «бумажная» прибыль |

Зелёные флаги:

| Сигнал | Что значит |
|--------|-----------|
| Стабильный рост выручки > инфляции | Реальный рост бизнеса |
| Маржа EBITDA стабильна или растёт | Масштаб работает |
| FCF > чистой прибыли | Качественная прибыль |
| Net Debt/EBITDA < 1.5x | Низкий долг, запас для дивидендов |
| ROE > 20% устойчиво | Качественный бизнес |
| OCF / Net Income > 0.8 | Качественная, «денежная» прибыль |
| Buyback + дивиденды > 5% от капитализации | Высокий возврат акционерам |

**Учитывай сезонность:**

Для секторов с выраженной сезонностью (ритейл, строительство, агро, туризм) квартальные показатели сравнивай **YoY** (год к году), а не QoQ (квартал к кварталу). Падение выручки Q1 vs Q4 в ритейле — норма, а не сигнал.

### 4а. Проверь stop-условия

Перед расчётом справедливой цены убедись, что анализ вообще имеет смысл. Если выполняется **хотя бы одно** из условий ниже — прекрати анализ, position = `avoid`:

| Условие | Почему stop |
|---------|-------------|
| Нет МСФО-отчётности > 2 лет | Невозможно оценить — нет данных для мультипликаторов |
| ADV < 1 млн ₽ | Крайняя неликвидность — невозможно войти/выйти без существенного impact на цену |
| Компания в SDN-листе OFAC | Полная санкционная блокировка — риск заморозки активов, невозможность расчётов |
| Идёт процедура банкротства или объявлен делистинг | Акционерный капитал может быть обнулён |
| Free-float < 5% и нет планов на увеличение | Цена неинформативна, манипулируема, выход из позиции практически невозможен |

Если stop-условие сработало — запиши причину в карточку компании (`_index.md`) в секцию «Мой тезис» и поставь `position: avoid`, `sentiment: bearish`.

### 5. Определи справедливую цену

Целевой мультипликатор выводится из формулы, а не угадывается. Но **выбор основного метода зависит от долговой нагрузки**.

**Единая шкала долговой нагрузки** (используется во всех секциях документа):

| Net Debt/EBITDA | Уровень | Влияние на оценку |
|-----------------|---------|-------------------|
| < 1,5x | Низкий | Долг не влияет, поправка 0% |
| 1,5–2,5x | Умеренный | Поправка −10% к P/E |
| 2,5–3,5x | Высокий | Переходи на EV/EBITDA, поправка −25% к P/E |
| > 3,5x | Критический | P/E не применим, только EV/EBITDA + стресс-тест |

Выбор основного метода:

| Net Debt/EBITDA | Основной метод | Почему |
|-----------------|---------------|--------|
| < 2,5x          | P/E | Долг невысок, прибыль акционера стабильна |
| 2,5–3,5x        | EV/EBITDA | Долг существенный — процентные расходы нелинейно растут при повышении ставки, прибыль волатильна |
| > 3,5x          | EV/EBITDA + стресс-тест | Высокий риск — при росте ставки на 2-3п.п. обслуживание долга может съесть всю прибыль |

**Почему P/E опасен для закредитованных компаний:** P/E оценивает остаточный поток акционеру (после процентов). При долге 3,5x EBITDA и ставке 20% процентные расходы ≈ 70% EBITDA. Рост ставки на 2п.п. увеличивает расходы на ~7% EBITDA, но для акционера это может быть -30% прибыли. Эффект рычага делает прибыль крайне чувствительной к ставке, а P/E — ненадёжным.

#### Trailing vs Forward мультипликаторы

| Тип | Когда использовать |
|-----|-------------------|
| **Trailing (LTM)** | Зрелые компании со стабильной прибылью; когда нет надёжного прогноза |
| **Forward (NTM)** | Растущие компании; после разового события (списание, курсовая разница); когда есть консенсус-прогноз аналитиков |

Для растущих компаний trailing P/E завышает оценку — используй forward P/E на прогнозную прибыль. Консенсус-прогнозы бери у брокеров (BCS, SberCIB) или строй сам на основе guidance менеджмента.

#### Шаг 1: Рассчитай базовые метрики

```
EPS = Чистая прибыль (LTM или прогноз NTM) / Кол-во акций
EBITDA per share = EBITDA (LTM или прогноз NTM) / Кол-во акций
Net Debt per share = Чистый долг / Кол-во акций
```

Кол-во акций — со smart-lab (`https://smart-lab.ru/q/{TICKER}/f/y/`), строка «Число акций ао». **Учитывай buyback:** если компания выкупает акции, EPS растёт даже при стабильной прибыли.

#### Шаг 2: Определи целевой P/E

```
Целевой P/E = 1 / (r − g) × Поправка за качество
```

Где:
- **r** = ставка ЦБ + премия за риск акций (ERP). ERP для российского рынка ≈ 5%
- **g** = устойчивый темп роста прибыли компании. Оценивай по CAGR выручки за 3-4 года, скорректированному на тренд маржи. Если маржа стабильна — g ≈ рост выручки. Если маржа сжимается — g < рост выручки

**Как выбрать g:**
- Посмотри CAGR выручки за 3-4 года (номинальный, с инфляцией)
- Проверь тренд маржи EBITDA — если падает, скорректируй g вниз
- Для зрелых компаний: g ≈ 3-5% (инфляция)
- Для растущих: g ≈ 8-12%
- Для циклических: g = средний за цикл

**Ограничения g:**
- **Жёсткий потолок: g ≤ 12%.** Даже самая быстрорастущая компания не может расти темпом 15-20% бесконечно. Формула 1/(r−g) описывает бессрочную оценку, поэтому g — это долгосрочный устойчивый рост, а не текущий CAGR
- **g не может быть ≥ r.** Если r=21% и g=15%, знаменатель = 6%, P/E = 17x — уже на грани разумного. При g → r формула даёт P/E → ∞, что бессмысленно
- **Минимальный спред (r−g) ≥ 5%.** Если спред меньше — оценка слишком чувствительна к допущениям: изменение g на 1п.п. при спреде 5% меняет P/E на ~25%, а при спреде 3% — на ~50%. При спреде < 5% пересмотри g в сторону снижения
- Если текущий CAGR выручки компании 20%+, а g ограничен 12% — это нормально. Высокий текущий рост учитывается через поправку за качество (+10%), а не через g
- **g может быть нулевым или отрицательным.** Для declining-бизнесов (стагнация выручки, структурное падение спроса) g = 0% или от −1% до −3%. При g < 0 формула 1/(r−g) работает корректно: знаменатель увеличивается, P/E снижается (например, r=21%, g=−2% → P/E = 1/0.23 = 4,3x)
- **Для циклических компаний** g = средний CAGR за полный цикл (5–7 лет), а не за восходящую фазу. Брать g с верхней точки цикла — типичная ошибка, завышающая оценку
- **Формат подстановки:** r и g подставляются в формулу как десятичные дроби: r = 0.21 (не 21), g = 0.07 (не 7). Пример: 1/(0.21 − 0.07) = 1/0.14 = 7,1x

#### Шаг 3: Поправка за качество

Системная корректировка вместо «на глаз». Чеклист:

| Фактор | Критерий | Поправка |
|--------|----------|----------|
| ROE > среднего по сектору | Сравни с таблицей из `sectors/{sector}/_index.md` | +10% |
| Рост выручки > сектора | Аналогично | +10% |
| Маржа EBITDA падает 3+ кварталов | Устойчивая потеря эффективности | -10% |
| Лидер сектора по масштабу | Переговорная сила, устойчивость | +10% |
| Зависимость от 1-2 клиентов / госзаказа | Концентрация риска | -10% |

**Долговая нагрузка** — отдельная шкала (риск долга нелинеен, плоская поправка -10% недостаточна):

| Net Debt/EBITDA | Поправка к P/E | Действие |
|-----------------|---------------|----------|
| < 1,5x | 0% | Долг не влияет |
| 1,5-2,5x | -10% | Умеренный долг |
| 2,5-3,5x | -25% | Переходи на EV/EBITDA как основной метод (шаг 3а) |
| > 3,5x | **P/E не применим** | Только EV/EBITDA + стресс-тест |

Почему нелинейно: при долге 3x и ставке 20% процентные расходы ≈ 60% EBITDA. Рост ставки на 2п.п. добавляет ~6% EBITDA к расходам, но для акционера это может быть -30% чистой прибыли. Эффект рычага усиливает влияние ставки на прибыль.

Суммируй поправки из чеклиста + долговую поправку. Итог — множитель к базовому мультипликатору. Пример: +10% качество, -10% долг → ×1,00.

#### Шаг 3а: EV/EBITDA для закредитованных компаний

Если Net Debt/EBITDA > 2,5x — используй EV/EBITDA как основной метод вместо P/E.

```
Целевой EV/EBITDA = (1 − tax) / (WACC − g) × Поправка за качество
```

Где:
- **tax** ≈ 20-25% (эффективная ставка налога на прибыль в РФ)
- **WACC** = r_equity × E/(D+E) + r_debt × (1−tax) × D/(D+E)
  - r_equity = ставка ЦБ + ERP (как в P/E методе)
  - r_debt ≈ ставка ЦБ + 1-3% (кредитный спред, зависит от рейтинга)
  - E/(D+E) и D/(D+E) — доли equity и долга в EV (бери текущие)
- **g** — те же правила и ограничения, что для P/E

```
Целевая цена = (EBITDA × Целевой EV/EBITDA − Net Debt) / Кол-во акций
```

**Стресс-тест (обязателен при > 3,5x):** рассчитай процентные расходы при ставке ЦБ +3п.п. от текущей. Если они превышают 50% EBITDA — красный флаг, компания может не пережить ужесточение ДКП. В этом случае ставь sentiment `bearish` независимо от upside.

#### Шаг 3б: Дисконт за госуправление (GOD — Government Ownership Discount)

Если государство владеет >50% компании (напрямую или через госструктуры), применяется дисконт к целевой цене. Госкомпании систематически недоплачивают миноритариям: низкий payout, непрозрачные инвестпрограммы, менеджмент без KPI на капитализацию.

**Базовый дисконт:**

| Критерий | Дисконт | Логика |
|----------|---------|--------|
| Гос. владение >50%, payout < 30% | **-30%** | Деньги не доходят до миноритариев |
| Гос. владение >50%, payout 30-50% | **-20%** | Частичный возврат, но ниже рыночного |
| Гос. владение >50%, payout > 50% | **-10%** | Государство научилось делиться |
| Частная компания | **0%** | Дисконт не применяется |

**Смягчающие/отягчающие факторы (поверх базового):**

| Фактор | Коррекция | Пример |
|--------|-----------|--------|
| Казначейские акции >20% без плана гашения | **-5%** | IRAO (29,5% казначейских) |
| Анонсирован рост payout / buyback | **+5%** | Если менеджмент объявил планы |
| Менеджмент = чиновники без KPI на капитализацию | **-5%** | Типично для госмонополий |
| История стабильного роста дивидендов >3 лет | **+5%** | Сбер, IRAO |

**Итого дисконт** = базовый + коррекции, ограничен диапазоном **[-40%, 0%]**.

**Применение:**

```
Скорректированная целевая цена = Целевая цена × (1 + дисконт_за_госуправление)
```

Пример — IRAO: payout 25% → базовый -30%; казначейские 29,5% без плана гашения → -5%; стабильный рост дивидендов >3 лет → +5%. Итого: **-30%**. Fair value: 7,0 × 0,70 = 4,9 ₽.

**YAML-поля для карточки компании:**
- `gov_ownership`: доля государства (0-100, или `0` если частная)
- `gov_discount`: применённый дисконт (напр. `-0.30` для -30%)

#### Шаг 3в: Особые типы компаний

Стандартные методы (P/E, EV/EBITDA) не работают для некоторых типов бизнеса. Определи тип компании **до** расчёта мультипликаторов и используй соответствующий подход.

**Банки и страховые:**

| Параметр | Значение |
|----------|----------|
| Основной метод | P/BV × ROE (Justified P/BV = ROE / (r − g)) |
| Net Debt/EBITDA | **Не применим** — долг = основа бизнеса банка |
| EV/EBITDA | **Не применим** — EBITDA не имеет смысла для финансовых компаний |
| Специфические метрики | NIM (чистая процентная маржа), Cost/Income, NPL ratio (доля просроченных кредитов), достаточность капитала (Н1.0) |
| Поправка за качество | Те же факторы (ROE, лидерство, госуправление) + стоимость риска, NPL, достаточность капитала |

Формула: `Целевая цена = BV per share × Justified P/BV × Поправка за качество`

**Холдинги (АФК Система, Эталон):**

| Параметр | Значение |
|----------|----------|
| Основной метод | NAV (Net Asset Value) — сумма рыночной стоимости долей в дочках минус чистый долг холдинга |
| Стандартный дисконт к NAV | 20–40% (непрозрачность структуры, корпоративные расходы, конфликт интересов) |
| Альтернатива | P/E по сумме долей — если все дочки публичные |
| Когда дисконт ниже (20%) | Прозрачная структура, анонсированные планы по раскрытию стоимости (IPO дочек, spin-off) |
| Когда дисконт выше (40%) | Высокий долг холдинга, непубличные дочки, история размытия миноритариев |

Формула: `Целевая цена = (Σ Доля × Стоимость дочки − Net Debt холдинга) × (1 − дисконт) / Кол-во акций`

**Компании с отрицательной прибылью (OZON, стартапы):**

| Параметр | Значение |
|----------|----------|
| Основной метод | EV/Sales с бенчмарком по сектору (сравни с прибыльными аналогами, скорректируй на маржу) |
| P/BV | Применим, если есть значимые материальные активы (недвижимость, запасы) |
| DCF на прогнозную прибыль | Применим, если есть видимость выхода в прибыль (guidance менеджмента, тренд маржи) |
| P/E, дивидендный метод | **Не применимы** — нет прибыли, нет дивидендов |
| Ключевой вопрос | Когда компания выйдет на безубыточность по EBITDA? Если горизонт > 3 лет — высокий риск |

Формула: `Целевая цена = (Sales × Целевой EV/Sales − Net Debt) / Кол-во акций`

#### Шаг 4: Три сценария

Считай целевую цену для трёх уровней ставки ЦБ. Прогноз ставки бери из `russia/macro.md`.

Для P/E (Net Debt/EBITDA < 2,5x):

| | Пессимистичный | Базовый | Оптимистичный |
|---|---|---|---|
| Ставка ЦБ (горизонт 12 мес.) | текущая (без снижения) | прогноз ЦБ | агрессивное снижение |
| r = ставка + ERP | | | |
| Базовый P/E = 1/(r−g) | | | |
| Целевой P/E (×поправка) | | | |
| **Целевая цена = EPS × P/E** | | | |
| **Upside** | | | |

Для EV/EBITDA (Net Debt/EBITDA > 2,5x) — аналогичная таблица с WACC вместо r и EV/EBITDA вместо P/E.

#### Шаг 5: Чувствительность к g

g — самое субъективное допущение. Покажи, как меняется целевая цена при разных g (в базовом сценарии ставки):

| g | Целевой P/E | Цена | Upside |
|---|-------------|------|--------|
| g − 3п.п. | | | |
| **g (базовый)** | | | |
| g + 2п.п. | | | |

Если при g−3п.п. upside уходит в минус — тезис хрупкий.

#### Шаг 6: Перекрёстная проверка

Два дополнительных метода — для проверки, не как основа.

**DCF (упрощённый):**

```
Стоимость = Σ(FCF_t / (1+r)^t) + Terminal Value / (1+r)^n
```

- FCF — нормализованный (не аномальный LTM). Если FCF LTM сильно отличается от прибыли — используй прибыль × исторический коэффициент конверсии
- r — из базового сценария
- Прогнозный период — 5 лет, рост = g
- Terminal Value = FCF_5 × (1 + g_terminal) / (r − g_terminal), где g_terminal = 3-5%

**Дивидендный:**

```
Целевая цена = DPS (прогноз) / Целевая дивдоходность
```

- Целевая дивдоходность = r − g (часть доходности через рост, часть через дивиденды)
- Ненадёжен для компаний с нестабильной дивидендной политикой или недавним началом выплат

Если DCF и P/E дают близкие значения — оценка устойчива. Если расходятся >30% — пересмотри допущения.

#### Итог

**`my_fair_value` = целевая цена из базового сценария (шаг 4).** Это справедливая стоимость при реализации прогноза ЦБ по ставке и базового g.

### 6. Рассчитай upside и прими решение

```
Upside = (my_fair_value − current_price) / current_price × 100%
```

| Upside | Position | Когда |
|--------|----------|-------|
| > 30% | **buy** | Значительная недооценка + понятный катализатор |
| 15-30% | **watch** | Интересно, но ждём лучшую точку входа или подтверждения |
| 0-15% | **hold** | Справедливо оценена (если уже в портфеле — держать) |
| < 0% | **sell** | Переоценена |

Одного upside недостаточно. Дополнительные критерии:

- **Катализатор** — что заставит рынок переоценить компанию? (отчётность, дивиденды, снижение ставки, M&A, buyback)
- **Margin of safety** — лучше пропустить сделку, чем купить без запаса. Если расчёты дают upside 15%, но допущения агрессивные — это не покупка
- **Качество бизнеса** — дешёвая компания с падающей маржой и растущим долгом может стать ещё дешевле (value trap)

## Признаки недооценённой компании

Компания может быть недооценена, если выполняется **комбинация** (не один пункт):

1. **Мультипликаторы ниже исторических** — P/E или EV/EBITDA ниже собственного среднего за 3-5 лет
2. **Дисконт к сектору при сопоставимом качестве** — P/E ниже среднего по сектору, при этом ROE и рост выручки не хуже
3. **Рост не учтён в цене** — выручка и прибыль растут, а мультипликаторы стоят на месте или сжимаются
4. **Разовый негатив** — падение цены из-за временного фактора (разовый убыток, негативная новость), но бизнес не пострадал
5. **Предстоящий катализатор** — скоро выйдет сильная отчётность, будут объявлены дивиденды, ожидается снижение ставки

Компания НЕ недооценена, даже если дешёвая по мультипликаторам, если:
- Прибыль/выручка стабильно падают (value trap)
- Высокий долг при высокой ставке и нет плана по снижению
- Отрасль в стадии decline без катализатора
- Корпоративное управление вызывает вопросы (размытие, сделки с аффилированными лицами)

## Чеклист перед buy

Перед тем как поставить position = `buy`, убедись:

- [ ] **Upside > 30%** в базовом сценарии
- [ ] **Катализатор есть** — что заставит рынок переоценить? (отчёт, дивиденды, снижение ставки)
- [ ] **Тезис записан** — можешь объяснить идею в 2-3 предложениях
- [ ] **Net Debt/EBITDA < 2.5x** или есть план по снижению долга
- [ ] **Маржа стабильна или растёт** — не падает 3+ квартала подряд
- [ ] **Ликвидность достаточна** — ADV > 10 млн ₽
- [ ] **Качество прибыли** — OCF/Net Income > 0.5
- [ ] **Геополитика учтена** — санкционный статус, импортозависимость (см. `russia/geopolitics.md`)
- [ ] **Госуправление учтено** — если гос. владение >50%, применён GOD (шаг 3б)
- [ ] **Нет value trap признаков** (см. ниже)

Если хотя бы 2 пункта не выполнены — пересмотри решение или снизь до `watch`.

## Примеры value trap (антипаттерны)

**Признаки value trap — когда «дешёвая» компания становится ещё дешевле:**

| Паттерн | Пример | Почему ловушка |
|---------|--------|----------------|
| Падающая отрасль | Угольные компании с P/E 3x | Спрос структурно снижается, прибыль будет падать |
| Цикл на пике | Металлурги в 2021 с рекордной прибылью | P/E низкий, потому что прибыль аномально высокая — при нормализации мультипликатор вырастет, а цена упадёт |
| Высокий долг + высокая ставка | Застройщики с Net Debt/EBITDA 4x | Процентные расходы съедают прибыль, риск дефолта |
| Разовая прибыль | Компания продала актив, P/E = 2x | Следующий год — P/E 10x на нормальной прибыли |
| Плохое корпоративное управление | Допэмиссия, сделки с аффилированными лицами | Мажоритарий извлекает value в ущерб миноритариям |

**Правило:** если компания дешёвая по мультипликаторам, но прибыль/выручка падают 2+ года — это не недооценка, это рынок закладывает дальнейшее ухудшение.

## Чего НЕ делать

- Не выбирай целевой P/E «на глаз» или «исторический средний» — используй формулу 1/(r−g) со сценариями по ставке ЦБ
- Не покупай только потому что «P/E низкий» — всегда проверяй почему
- Не игнорируй макро: при ставке ЦБ 16% депозит даёт 16% без риска — акция должна предлагать больше
- Не экстраполируй один хороший квартал на всю компанию — смотри тренд за 3-4 года
- Не забывай про долг: компания с Net Debt/EBITDA > 3,5x и ставкой 16% тратит огромную часть прибыли на обслуживание
- Не ставь sentiment `bullish` без заполненной целевой цены и тезиса
- Не ставь buy для госкомпании с payout <30% без экстремального upside (>60% после GOD-дисконта)
- Не оставляй HTML-комментарии `<!-- -->` из шаблона в заполненном документе

## Пример расчёта: Сбербанк (SBER)

Сквозной пример применения методологии (данные условные, для иллюстрации).

**Исходные данные (LTM):**
- Чистая прибыль: 1 500 млрд ₽
- Кол-во акций: 21,6 млрд шт.
- EPS = 1500 / 21,6 = **69,4 ₽**
- Текущая цена: 270 ₽
- Trailing P/E = 270 / 69,4 = **3,9x**
- ROE: 24%
- Net Debt/EBITDA: не применимо (банк)

**Шаг 2: Целевой P/E**

Параметры:
- Ставка ЦБ (базовый сценарий): 18%
- ERP: 5%
- r = 18% + 5% = **23%**
- g (рост прибыли): банк зрелый, рост ≈ номинальный ВВП = **7%**
- Спред r − g = 16% ✓ (> 5%, ок)

```
Базовый P/E = 1 / (0.23 − 0.07) = 1 / 0.16 = 6,25x
```

**Шаг 3: Поправка за качество**
- ROE 24% > среднего по сектору (18%): **+10%**
- Лидер сектора по масштабу: **+10%**
- Санкционные риски: **-10%**
- Итого поправка: +10%

```
Целевой P/E = 6,25 × 1,10 = 6,9x
```

**Шаг 4: Целевая цена**

```
Целевая цена = EPS × Целевой P/E = 69,4 × 6,9 = 479 ₽
```

**Шаг 6: Upside**

```
Upside = (479 − 270) / 270 = 77%
```

**Вывод:** Upside > 30%, катализатор — снижение ставки ЦБ и дивиденды. Position = **buy**.

**Чувствительность:**

| Сценарий | Ставка ЦБ | r | P/E | Цена | Upside |
|----------|-----------|---|-----|------|--------|
| Пессимистичный | 21% | 26% | 5,8x | 403 ₽ | +49% |
| **Базовый** | 18% | 23% | 6,9x | 479 ₽ | +77% |
| Оптимистичный | 14% | 19% | 9,2x | 638 ₽ | +136% |

Даже в пессимистичном сценарии upside > 30% — тезис устойчив.
