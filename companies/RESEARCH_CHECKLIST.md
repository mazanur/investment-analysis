## Фаза 5: Определение справедливой цены

### Шаг 5.1: Определить тип компании

- [ ] **Стандартная** → P/E или EV/EBITDA (см. ниже)
- [ ] **Банк/страховая** → P/BV × ROE (Justified P/BV = ROE / (r − g)). Net Debt/EBITDA и EV/EBITDA не применимы
- [ ] **Холдинг** → NAV (сумма долей − долг холдинга) × (1 − дисконт 20-40%)
- [ ] **Убыточная** → EV/Sales с бенчмарком по сектору

### Шаг 5.2: Выбрать основной метод (для стандартных)

- [ ] Net Debt/EBITDA < 2,5x → **P/E** (основной)
- [ ] Net Debt/EBITDA 2,5–3,5x → **EV/EBITDA** (основной)
- [ ] Net Debt/EBITDA > 3,5x → **EV/EBITDA + стресс-тест** (обязателен)

### Шаг 5.3: Рассчитать базовые метрики

- [ ] EPS = Чистая прибыль (LTM или NTM) / Кол-во акций
- [ ] EBITDA per share = EBITDA / Кол-во акций (если EV/EBITDA)
- [ ] Net Debt per share = Чистый долг / Кол-во акций
- [ ] BV per share = Балансовая стоимость / Кол-во акций (если банк)
- [ ] Trailing vs Forward? Если растущая компания или разовое событие → Forward (NTM)

### Шаг 5.4: Определить r и g

- [ ] **r** = ставка ЦБ (из `macro.md`, базовый сценарий) + ERP (5%)
- [ ] **g** = CAGR выручки за 3-4 года, скорректированный на тренд маржи
  - Зрелые: 3-5% | Растущие: 8-12% | Циклические: средний за цикл | Declining: 0% или отрицательный
- [ ] Проверить ограничения:
  - g ≤ 12% (жёсткий потолок)
  - g < r (иначе формула бессмысленна)
  - (r − g) ≥ 5% (иначе оценка нестабильна)
- [ ] Если WACC нужен (EV/EBITDA): WACC = r_equity × E/(D+E) + r_debt × (1−tax) × D/(D+E)

### Шаг 5.5: Рассчитать базовый мультипликатор

- [ ] **P/E метод:** базовый P/E = 1 / (r − g). Формат: r и g как десятичные дроби (0.21, не 21)
- [ ] **EV/EBITDA метод:** базовый EV/EBITDA = (1 − tax) / (WACC − g)

### Шаг 5.6: Поправка за качество

- [ ] ROE > среднего по сектору → +10%
- [ ] Рост выручки > сектора → +10%
- [ ] Маржа EBITDA падает 3+ кварталов → -10%
- [ ] Лидер сектора по масштабу → +10%
- [ ] Зависимость от 1-2 клиентов / госзаказа → -10%
- [ ] Долговая нагрузка (нелинейная шкала):
  - ND/EBITDA < 1.5x → 0%
  - ND/EBITDA 1.5-2.5x → -10%
  - ND/EBITDA 2.5-3.5x → -25%
  - ND/EBITDA > 3.5x → P/E не применим
- [ ] **Итого поправка** = сумма факторов. Целевой мультипликатор = базовый × (1 + итого поправка)

### Шаг 5.7: GOD-дисконт (если гос. владение > 50%)

Данные для расчёта — из `governance.md`.

- [ ] Определить базовый дисконт по payout:
  - payout < 30% → -30%
  - payout 30-50% → -20%
  - payout > 50% → -10%
- [ ] Коррекции:
  - Казначейские > 20% без гашения → -5%
  - Анонсирован рост payout / buyback → +5%
  - Менеджмент-чиновники без KPI → -5%
  - Стабильный рост дивидендов > 3 лет → +5%
- [ ] **Итого GOD** = базовый + коррекции (ограничен [-40%, 0%])
- [ ] Записать в `governance.md` → таблица «Расчёт GOD-дисконта»

### Шаг 5.8: Стресс-тест (если ND/EBITDA > 3,5x)

- [ ] Рассчитать процентные расходы при ставке ЦБ +3п.п.
- [ ] Процентные расходы > 50% EBITDA? → `sentiment: bearish` (независимо от upside)

### Шаг 5.9: Три сценария по ставке ЦБ

- [ ] **Пессимистичный** — текущая ставка (без снижения)
- [ ] **Базовый** — прогноз ЦБ (из `macro.md`)
- [ ] **Оптимистичный** — агрессивное снижение
- [ ] Для каждого: r → базовый мультипликатор → целевой мультипликатор → целевая цена → upside
- [ ] Применить GOD-дисконт к целевой цене каждого сценария (если применимо)

### Шаг 5.10: Чувствительность к g

- [ ] g − 3п.п. → целевой мультипликатор → цена → upside
- [ ] g (базовый) → целевой мультипликатор → цена → upside
- [ ] g + 2п.п. → целевой мультипликатор → цена → upside
- [ ] При g−3п.п. upside < 0%? → тезис хрупкий, учесть при определении position

### Шаг 5.11: Перекрёстная проверка

- [ ] **DCF (упрощённый):** FCF × 5 лет (рост = g) + Terminal Value. Если FCF аномальный — нормализовать
- [ ] **Дивидендный:** DPS / (r − g). Ненадёжен для нестабильных выплат
- [ ] DCF и основной метод расходятся > 30%? → пересмотреть допущения

### Шаг 5.12: Итог

- [ ] **my_fair_value** = целевая цена из базового сценария (шаг 5.9)
- [ ] Записать значение

## Фаза 6: Решение

### Upside и position

- [ ] `upside = (my_fair_value − current_price) / current_price × 100%`
- [ ] Определить position:
  - &gt; 30% → **buy** (+ катализатор обязателен)
  - 15-30% → **watch**
  - 0-15% → **hold**
  - &lt; 0% → **sell**

### Чеклист перед buy (если position = buy)

- [ ] Upside > 30% в базовом сценарии
- [ ] Катализатор есть (конкретный, с датой) — из `events.md`
- [ ] Тезис записан (можно объяснить в 2-3 предложениях)
- [ ] Net Debt/EBITDA < 2.5x (или план по снижению)
- [ ] Маржа стабильна или растёт (не падает 3+ квартала)
- [ ] ADV > 10 млн ₽
- [ ] Качество прибыли OCF/NI > 0.5
- [ ] Геополитика учтена — санкционный статус из `governance.md`
- [ ] GOD учтён (если гос. владение > 50%)
- [ ] Нет признаков value trap (падающая выручка/прибыль 2+ года, цикл на пике, разовая прибыль)

> Если 2+ пункта не выполнены → снизить до **watch**.

### Проверка по консенсусу

- [ ] Проверить `sources/broker_favorites_2026.md` — сколько брокеров рекомендуют
  - 8+ подтверждает bullish
  - 0 — red flag, проверить почему рынок не согласен
- [ ] Если наша оценка сильно расходится с консенсусом → пересмотреть допущения

### Финальное решение

- [ ] Определить `sentiment`: bullish / neutral / bearish
- [ ] Определить `position`: buy / watch / hold / sell / avoid
- [ ] Сформулировать тезис (3-5 предложений): почему именно этот position, ключевой аргумент, катализатор, горизонт
