# Корпоративное управление: Название (TICKER)

Данные для расчёта GOD-дисконта и оценки рисков корпоративного управления.
Автоматические секции обновляются: `make download-governance TICKER=TICKER && make fill-governance TICKER=TICKER`.

## Как обновить

### Автоматически (MOEX ISS + OpenSanctions)

Секции «Дивидендная история» и «Санкционный скрининг» заполняются автоматически:

```bash
# 1. Скачать данные (если ещё нет)
make download-events TICKER=TICKER      # дивиденды с MOEX
make download TICKER=TICKER             # финансы со smart-lab (для payout ratio)
make download-governance TICKER=TICKER  # санкционный скрининг

# 2. Сгенерировать governance.md
make fill-governance TICKER=TICKER

# Или для всех компаний сразу:
make download-all && make fill-governance
```

Скрипт использует:
- **Дивиденды MOEX** → история выплат, стабильность, периодичность
- **Smart-lab CSV** → payout ratio (если скачан)
- **OpenSanctions API** → санкционный скрининг

Данные сохраняются в `data/moex_events.json`, `data/smartlab_yearly.csv`, `data/sanctions.json`.

### Вручную (секции ниже)

Шесть секций требуют ручного заполнения — они **не перезаписываются** при повторном запуске:
- **Структура акционеров** — IR-страница, e-disclosure.ru, smart-lab
- **Дивидендная политика** — текст политики из устава / решения СД
- **Программа buyback** — пресс-релизы, существенные факты
- **Менеджмент** — годовой отчёт, IR-страница
- **Риски корпоративного управления** — чек-лист рисков
- **Расчёт GOD-дисконта** — заполняет Claude на основе данных выше
