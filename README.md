## Как использовать

### С Claude Code

Проект оптимизирован для работы с [Claude Code](https://claude.ai/claude-code). При запуске в директории проекта Claude автоматически читает `CLAUDE.md` и понимает структуру базы знаний.

Примеры запросов:
- «Заполни сектор нефтегаза» — Claude проведёт веб-исследование и создаст `sectors/oil-gas/_index.md`
- «Проанализируй Сбербанк» — Claude соберёт данные со smart-lab, рассчитает целевую цену, заполнит `companies/SBER.md`
- «Что думаешь про X5?» — Claude прочитает карточку и даст оценку с учётом секторального и макро-контекста

### Без Claude

Можно использовать как обычную базу знаний в Markdown:
1. Начни с `_index.md` — там навигация и статус всех документов
2. Для анализа компании читай снизу вверх: компания → сектор → макро
3. Шаблоны и инструкции в `sectors/_TEMPLATE.md` и `companies/RESEARCH_GUIDE.md`

## Методология оценки

### Секторальный анализ

Полная инструкция: [`sectors/RESEARCH_GUIDE.md`](sectors/RESEARCH_GUIDE.md)

- Стадия цикла (growth / mature / declining / cyclical)
- Драйверы роста и риски
- Влияние макро-факторов (ставка ЦБ, курс, инфляция)
- Критерии отбора компаний для инвестирования

### Оценка компаний

Полная инструкция: [`companies/RESEARCH_GUIDE.md`](companies/RESEARCH_GUIDE.md)

#### Простое объяснение

**Таргет = сколько компания должна стоить, если рынок оценит её справедливо.**

1. Смотрим, сколько компания зарабатывает (прибыль или EBITDA)
2. Считаем справедливый мультипликатор из формулы
3. Корректируем за риски и качество
4. Умножаем прибыль на мультипликатор = стоимость компании
5. Делим на количество акций = таргет

Если текущая цена сильно ниже таргета (upside > 30%) — компания недооценена.

#### Какой метод использовать

| Ситуация | Метод |
|----------|-------|
| Долг < 2.5x EBITDA | P/E |
| Долг > 2.5x EBITDA | EV/EBITDA |
| Банк или страховая | P/BV |

#### Формулы

**P/E (для компаний с маленьким долгом):**
```
Справедливый P/E = 1 / (требуемая доходность − рост)
Таргет = Прибыль на акцию × P/E
```

**EV/EBITDA (для компаний с большим долгом):**
```
Справедливый EV/EBITDA = (1 − налог) / (WACC − рост)
Таргет = (EBITDA × EV/EBITDA − Долг) / Кол-во акций
```

**Требуемая доходность** = прогнозная ставка ЦБ + 5% (премия за риск акций)

#### Поправки за качество

Формула даёт базовый мультипликатор. Дальше корректируем:

| Фактор | Поправка |
|--------|----------|
| ROE выше рынка (>15%) | +10% |
| Быстрый рост (>20%) | +10% |
| Высокий долг (>2.5x) | −10% до −25% |
| Нестабильный бизнес | −10% до −20% |
| Плохое корп. управление | −10% до −20% |

#### Пример: EUTR

- EBITDA = 24 млрд, Долг = 58 млрд, Акций = 159 млн
- Net Debt/EBITDA = 2.4x → используем EV/EBITDA
- Базовый EV/EBITDA = 0.80 / (0.148 − 0.07) = 10.3x
- После поправок = 9.3x
- EV = 24 × 9.3 = 223 млрд
- Equity = 223 − 58 = 165 млрд
- Таргет = 165 / 159 = **210 ₽** (с запасом безопасности)
- Текущая цена 139 ₽ → upside +52%

Три сценария по ставке ЦБ (пессимистичный / базовый / оптимистичный) показывают диапазон целевых цен и чувствительность к макро.
